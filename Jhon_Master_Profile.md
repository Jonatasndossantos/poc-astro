### ⏱️ Pitch de 3 Minutos ("The Hook")
*Use em: Abertura de Entrevista de Emprego ("Me fale sobre você").*

"Meu nome é Jônatas, moro em Extrema-MG, e tenho um histórico peculiar. Minha carreira começou fora da tecnologia, no McDonald's. Parece estranho mencionar isso numa entrevista técnica, mas foi lá que aprendi resiliência absoluta e execução sob extrema pressão – o que eu costumo chamar de meu 'treinamento SRE não-oficial'.

Enquanto a maioria usava o Laravel de forma tradicional, eu implantei Laravel Octane e Swoole para reter memória e quebrar gargalos críticos de velocidade. Paralelamente, desenvolvi do zero um motor de inteligência artificial proprietário consumindo a API da OpenAI, baseado em Strict JSON Schemas, para gerar contratos granulares perfeitamente formatados que reduziram semanas de trabalho burocrático para menos de 5 minutos.

Hoje, além de estar cursando Engenharia da Computação para construir minha base de baixo nível de forma imutável, eu atuo resolvendo *Race Conditions* em aplicações críticas e montando infraestruturas de Continuous Integration complexas. Sou muito focado no ecossistema PHP e React, e meu grande objetivo atual é escalar projetos em nível global."

### ⏱️ Pitch de 5 Minutos ("O Deep Dive de Negócios e Tecnologia")
*Use em: Entrevistas com Gestores de Engenharia (CTOs).*

Adicione ao final do pitch de 3 minutos:
"Como arquiteto, minha grande preocupação é resolver problemas de negócios sem criar complexidade acidental. Por exemplo, na Lumen, notei que a equipe sofria muito com alucinações das LLMs ao automatizar documentação pública de risco. Eu estanquei o problema não mudando o prompt, mas implementando RAG e OpenAI Function Calling no código, restringindo o LLM a um JSON Schema que obrigava a inteligência a responder e ler o banco MySQL da empresa em tempo real sem intermédio humano. 

Além disso, atuei na linha de frente resolvendo dívida técnica pesada (Refactoring). Quando os sistemas legados caíam por gargalo de *Race Conditions*, eu tinha total autonomia para acessar a máquina matriz em Linux via SSH, restaurar fluxos autônomos no n8n ou corrigir as lógicas de Observables no backend. Meu foco atual é dominar ambientes corporativos difíceis porque é lá onde eu prospero."

### ⏱️ Pitch de 10 Minutos ("Como Eu Resolvo Problemas – Estudo de Caso")
*Use em: "Me fale de um desafio técnico gigante que você resolveu." (Use o Framework S.T.A.R).*

**A Situação (S):**
Na LUMEN, nosso sistema para licitações públicas enfrentava grandes desafios em dois pilares. Um era processamento massivo; o outro era a morosidade e erro humano na geração de editais e contratos governamentais gigantescos (ETP, DFD). A dívida técnica do legado e problemas concorrentes ameaçavam o scale up da GovTech.

**A Tarefa (T):**
Fui encarregado, como Tech Lead e Arquiteto, de reconstruir e escalar o núcleo dessa infraestrutura enquanto estabilizava a aplicação herdada. Se o processo do governo travasse, o risco de compliance era altíssimo.

**A Ação (A):**
Tomei três decisões cruciais de arquitetura corporativa:
1. Fiz upgrade da topologia trocando o runtime convencional PHP. Implantei Laravel Octane acoplado ao Swoole/FrankenPHP. Isso transformou nossa infra num servidor rodando direto na RAM com estado retido, cortando latências assustadoras ao meio.
2. Construí um SaaS multi-tenant que conectava React/Inertia no painel administrativo a uma fila pesada no backend via *Queues* e *Workers*. Criei CI/CDs rigorosos no GitHub Actions para evitar build errors em produção.
3. No motor de IA, desenhei um pipeline com RAG nativo onde não usamos os chats convencionais, usamos a Assistant API forçando JSON objects (Function Calling). O sistema fragmentava o arquivo gigante do governo em "blocos editáveis" de Markdown estruturado. Assim, se a IA errasse um artigo do documento, o usuário refazia apenas aquelo minúsculo bloco ao invés do documento inteiro, poupando tokens (recursos financeiros do servidor) e tempo.

**O Resultado (R):**
O tempo de faturamento e geração de processos licitatórios caiu de *semanas para menos de cinco minutos*. Escalei um SaaS resiliente pra infraestrutura multi-tenant, mitiguei as *race conditions* no módulo de chat da aplicação legada e me posicionei como engenheiro de referência do núcleo criativo da empresa.  

### ⏱️ O Pitch de 1 Hora (Estrutura de "Conversa Técnica")
*Use em: Entrevistas Técnicas Abertas, Painéis com Arquitetos da Empresa. (Esta versão é na verdade o SEU mapa mental durante a conversa).*

- **15 Minutos Iniciais - Arquitetura de Software e Banco de Dados:** Você domina como o Laravel atua de forma opinativa gerando código previsível mas compreende porque às vezes precisa quebrar os anti-patterns (como N+1 Problems ou queries massivas usando Eager Loading e sub-queries). Argumentação forte no uso de SQL bem escrito.
- **15 Minutos - Pilar da Infra e DevOps:** Converse detalhadamente sobre como você interage com Linux remotamente. Deploy de servidores Nginx/Docker, lidando ativamente com logs nativos (`journalctl`, tailing) e troubleshooting crítico na "veia" do servidor via SSH. Mostre conhecimento do custo real da Cloud (como evitar pagar rios de dinheiro para a AWS quando um VPS cru resolve melhor).
- **15 Minutos - Pilar de Engenharia AI/R&D e Streaming:** Explique como RAG funciona de fato (Embeddings, Vector DB vs SQL, Temperature controls, Injeção de System Prompts engessados). Debata como WebSockets ou Server-Sent Events (SSE) são aplicados no front-end para evitar interfaces congeladas dando a impressão visual da stream de IA rodando.
- **15 Minutos - Visão de Cima (Decisões de Nível de CTO):** Argumente porque você escolheria o combo Blade + Livewire sobre um SPA complexo em React para 80% das aplicações (Speed To Market vs React Overengineering); e em contrapartida, mostre a inteligência de usar o Inertia.js (A ponte PHP+React) em casos em que o SPA é vital como em dashboards de análise governamental intensiva. Fale sobre a Engenharia da Computação e seu desejo implacável pela base estrutural.

## 🛠️ Parte 4: Matriz de Habilidades (As Palavras-Chave de Recrutador)
*Sempre que perguntarem "Você sabe/trabalha com X?", tenha essas respostas preparadas.*

#### 💻 Linguagens e Base Core
- **Desenvolvimento e OO:** PHP moderno avançado (Tipagem, Traits, Interfaces). JavaScript/TypeScript (Lidando com Event-Loops, Promises pesadas).
- **Bancos de Dados:** PostgreSQL, MySQL. Indexação correta, constraints e queries relacionais complexas que ferram o ORM.
- **Web Roots:** HTML Semântico, CSS Grid/Flexbox avançado (Lidando com responsividade nativa).

#### 🏗️ Backend, Frameworks & Alta Performance (Seu Foco Mais Forte)
- **Modern Laravel (11/12):** O coração da arquitetura. Você não usa só rotas; usa Queues pesadas, Events, Jobs assíncronos, Service Providers para abstração limpa.
- **Ecossistema de Alta Performance (Asynchronous PHP):** Laravel Octane, Swoole, FrankenPHP. Retenção de memória stateful dentro de um ecossistema stateless.
- **BaaS (Backend as a Service):** Supabase integrado rapidamente para testes ou produção imediata.

#### 🎨 Frontend, Ferramentas Visuais & Reatividade
- **Stack do Ecossistema Laravel:** TALL Stack (Tailwind, Alpine.js, Laravel, Livewire) sendo a arma secreta pra despachar ferramentas corporativas com coesão.
- **Stack de Interface Avulsa:** React 19, Hooks customizados e complexos. Inertia.js (a ponte perfeita) para um front-end hidratado limpo sem APIs expostas.
- **Styling Seguro e A11y:** Tailwind CSS v4 acoplado ao DaisyUI v5. Desenvolve temas dark/light sem destruir manutenção via plugins (`theme-controller`).

#### 🧠 R&D, Inteligência Artificial e Automação Corporativa
- **Engenharia de Prompt Nativa:** Integração avançada via OpenAI API (Assistant API moderna) usando Streaming e JSON Mode.
- **Tools / Function Calling:** Forçar o LLM a chamar funções da sua aplicação PHP (ex: *ir até o DB, rodar a soma mensal, e voltar isso pra uma string pro usuário*).
- **Integração Visual Legada via Node:** N8N rodando em background pra orquestração de APIs legadas sem toque de código bruto.

#### 🔨 DevOps, SRE e Cultura Shell
- **A Máquina Root:** Conexão remota e administração via SSH, segurança de chaves, Systemd, reverse proxying direto no Nginx (`etc/nginx/sites-available`).
- **Pipeline:** GitHub Actions para Continuous Integration blindando *bad commits* no servidor de produção.
- **Distribuição de Carga:** Otimização de Workers de filas em background. Resolve *Race Conditions* com lógica transacional e deadlocks.
